Роут запросов в браузере начинается с asgi.py
В этом файле мапируется тип соединения.

HTTP - get_asgi_application()
Эта функция сетапит джанго (сеттингс, юрл паттерны)
И возвращает ASGIHandler. Класс обслуживает только хттп соединения.
Распарсивает соединение.

WebSocket - коллится URLRouter, который подключает websocket_urlpatterns.

В паттернах задается урл для вебсокета. Через стандартные path(), re_path().
Вместо view, вторым параметром сюда передается Consumer.as_asgi()

В консумере происходит обработка всех действий. A.K.A. View.
Здесь при коннекте создаем имя группы комнаты и имя комнаты.
Имя комнаты получаем из scope. scope - словарь, 
	в котором есть словарь по ключу url_route
	в котором лежат позиционные и именованные аргументы
	Из именованных аргументов получаем 'room_name'.
	рум нейм судя по всему это паттерн из websocket_urlpatterns
В дальнейшем мы должны присоединиться к группе, передав в метод channel_layer.group_add()
параметры channel_name, room_group_name
В конце принять соединение сокета через .accept()
